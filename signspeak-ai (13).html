01
📷
Webcam Capture
Your browser captures a live video stream via WebRTC getUserMedia at up to 30 FPS. No video data ever leaves your device — all processing is 100% local. You can also upload a pre-recorded sign language video for offline analysis.
WebRTC · getUserMedia · File API
02
🖐️
MediaPipe Hand Detection
Each video frame is passed to Google's MediaPipe Hands model — a neural network compiled to WebAssembly that runs entirely in-browser. It detects up to 2 hands and maps 21 3D landmark points per hand including wrist, all knuckles, and all fingertips.
MediaPipe Hands · WASM · 21 landmarks × 2 hands
03
📐
Landmark Normalization
Raw pixel coordinates are normalized: centered on the wrist joint and scaled by the distance from wrist to middle-finger MCP. This makes the 63 features (x, y, z × 21) scale-invariant and position-invariant — the model works at any distance from the camera.
63 features · Scale-invariant · Position-invariant
04
🧠
TensorFlow.js Neural Network
The 63 normalized values are fed into a 3-layer dense neural network: 63 → Dense(128, ReLU) → Dropout(0.3) → Dense(64, ReLU) → Dropout(0.2) → Softmax(N classes). You train this model yourself in the Train tab with your own hand samples. Falls back to geometric finger-extension rules until a model is trained.
TF.js · Dense NN · Softmax · In-browser training
05
⏳
Hold-Frame Confirmation
A sign is only confirmed after the same prediction appears consistently for 8 consecutive frames (~0.25 seconds). This prevents false detections from brief hand movements between signs. A 2.4-second pause finalizes the current word.
8-frame threshold · Debounce filter
06
✦
Trigram Auto-Complete
After each confirmed word, a trigram language model predicts the 6 most likely next words based on your last 2 signed words. Covers common ASL conversational patterns — "HELLO HOW ARE", "I LOVE YOU", "THANK YOU PLEASE", etc. Tap any chip to insert instantly.
Trigram NLP · Context-aware · Click-to-insert
07
🤖
AI Sentence Builder
Your signed letters and words are sent to a language model which reconstructs them into grammatically correct, natural English sentences. It handles both letter-by-letter spelling (H-E-L-L-O → "Hello") and word-level input — producing fluent output from minimal signed input.
LLM · Natural language · Sentence reconstruction
08
🔊
Text-to-Speech Output
Every confirmed word and AI sentence is spoken aloud using the Web Speech API — built into all modern browsers with no plugin required. Choose from multiple voices and adjust speech rate from 0.5× to 2×.
Web Speech API · Multi-voice · Adjustable speed

About This Project
BRIDGE THE GAP

SignSpeak AI is an assistive technology project that uses computer vision and deep learning to make real-time ASL sign language translation accessible to everyone — with zero specialized hardware, no internet required, and no data leaving your device.

Built for education, healthcare, public services, and everyday communication, SignSpeak AI empowers deaf and hard-of-hearing individuals with a simple, browser-based translation tool backed by a real trainable neural network.

WHY WE BUILT THIS

Communication should have no barriers. SignSpeak AI makes it possible for deaf and hard-of-hearing individuals to be understood in any environment without an interpreter.

The Technology
WHAT POWERS SIGNSPEAK

SignSpeak AI combines Google's MediaPipe Hands (WASM-compiled neural network for real-time hand landmark detection), TensorFlow.js (in-browser training and inference of a dense classification network), a trigram NLP auto-complete system, and a large language model for natural sentence reconstruction — all running in a single HTML file with no build step, no server, and no dependencies to install.
